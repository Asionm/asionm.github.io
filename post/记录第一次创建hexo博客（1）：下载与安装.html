记录第一次创建hexo博客（1）：下载与安装

在一次分享会中，得知hexo配合github可以免费搭建一个博客。于是作为一个白嫖党的我，绝对不会错过这个机会的，于是就有了这个博客…….

实现原理

简介

HEXO

Hexo 是一款基于 Node.js 的静态博客框架。Hexo 使用 Markdown 解析文章，用户在本地安装Hexo并进行写作，通过一条命令，Hexo即可利用靓丽的主题自动生成静态网页。

参考HEXO官网

静态网页

在网站设计中，纯粹HTML（标准通用标记语言下的一个应用）格式的网页通常被称为“静态网页”，静态网页是标准的HTML文件，它的文件扩展名是.htm、.html，可以包含文本、图像、声音、FLASH动画、客户端脚本和ActiveX控件及JAVA小程序等。静态网页是网站建设的基础，早期的网站一般都是由静态网页制作的。静态网页是相对于动态网页而言，是指没有后台数据库、不含程序和不可交互的网页。静态网页相对更新起来比较麻烦，适用于一般更新较少的展示型网站。容易误解的是静态页面都是htm这类页面，实际上静态也不是完全静态，他也可以出现各种动态的效果，如GIF格式的动画、FLASH、滚动字幕等。

参考百度百科)

Github

GitHub是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库格式进行托管，故名GitHub。

参考百度百科

为什么HEXO+Github可以搭建个人博客？

因为HEXO是全静态的框架，所以也就意味着主要的呈现形式是ｈｔｍｌ,那么只需要把必要的 html js cs文件放到一个托管平台中，而这个托管平台可以向外访问这些文件，并且可以知道哪个是主页，那么就可以不经过服务器直接搭建出一个个人博客网页。而github 有一个pages功能，可以自定义主页，用来作为主页，然后再通过这个主页联系github中预先存的相关文件，一个网站就出来了。所以赶快白嫖起来

实现环境

ＷＩＮＤＯＷＳ

  • Git

到官网下载并安装Git，傻瓜式安装一直点即可

  • Node.js

下载安装Node.js，傻瓜式安装一直点即可

ＬＩＮＵＸ

  • 安装git
sudo apt-get install git
  • 安装完后输入下列命令查看是否安装完成
git -v
  • 安装node.js
sudo apt-get install nodejs
sudo apt-get install npm
  • 安装完后输入下列命令查看是否安装成功
node -v
npm -v

无论是WINDOWS还是LINUX都需要先拥有Github账号

注册地址：https://github.com/

实现步骤

安装HEXO

  • Windows用户需要创建一个博客文件夹，然后在文件夹中右键 git bash here（Linux用户从下一步开始）

​ 打开终端后输入下面的指令

npm install -g hexo-cli

​ 若一直报错说无法连接，可以尝试换源，换源的输入下面的指令（用的是淘宝源）

npm config set registry https://registry.npm.taobao.org

​ 安装完成后输入下面指令，查看是否安装成功

hexo -v

  • 初始化HEXO，输入下列指令
hexo init hexo-blog #其中hexo-blog可以换成你想要的名字，只是创建一个文件夹而已

​ 进入hexo-blog，就行安装，输入下面的指令

cd hexo-blog 	
npm install

​ 安装完成后可以输入下列指令来看看是否成功

hexo g && hexo s #更新hexo的内容以及启动本地服务

​ 然后打开http://localhost:4000/,查看是否有如下画面

​ 若有则成功安装

Github创建个人仓库

  • 进入github中，之前建立过库的右边会显示如下图

​ 新用户会显示如下

​ 无论是新用户还是旧用户均点击New

  • 点击New之后会出现如图界面，输入Repository name为你 asionm.github.io

​ 其中的asionm是我这个博客的名字，所以asionm需要替换成你的，输入完后点击下方的create repository

生成SSH密钥

  • 在博客文件夹右键点击git bash，然后属于下面的代码来配置git
git config --global user.name "yourname" #yourname改成你github的用户名
git config --global user.email "youremail" #youremail改成你注册github时用的邮箱

​ 设置完后，可以输入下面指令查看是否正确

git config user.name
git config user.email

​ 输入下面指令，生成SSH密钥

ssh-keygen -t rsa -C "youremail" #youremail改成你注册github时用的邮箱

​ 记录下密钥的地址，Windows一般在C:/users/你电脑的用户名/.ssh/

​ 地址下会看到三个文件

  • 打开id_rsa.pub文件复制里面所有内容，Ctrl A + Ctrl C。

​ 打开github->点击右上角下面的settings->SSH and GPG keys

​

​ 然后粘贴刚刚复制的内容到Key，Title随便填

  • 回到Git bash输入下面指令检查是否配置成功
ssh -T git@github.com

​ 若出现如下图所示信息则成功

HEXO部署到Github

  • 进入hexo-blog文件夹，打开_config.yml文件夹，若是Windows用户（也可以在bash中用vi编辑，i插入命令，esc->:wq保存并退出）可以直接记事本打开，在最下面复制粘贴如下内容
deploy:
  type: git
  repo: https://github.com/你的github名字/你的github名字.github.io.git
  branch: master

  • 然后安装deploy，输入下面指令
npm install hexo-deployer-git --save

  • 安装完成后输入下面的指令来推到github（国内经常会连接超时，建议挂梯子或者把github换成国内的gitee）
hexo clean #清理缓存，其实它清理的是原本生成的html js css等等文件
hexo generate #按照新的配置重新生成html js css等文件
hexo deploy #把生成的文件推到github

最后打开 yourgithubname.github.io 即可查看你的博客了
